What is HTML?

HTML (HyperText Markup Language) is the standard language for creating web pages. Think of it as the skeleton of a website — it gives structure to content.

How It Works

HTML uses tags (enclosed in < >) to mark up content. Most tags come in pairs — an opening tag and a closing tag:

<h1>Hello World!</h1>

What HTML Can Do

  • Structure content — headings, paragraphs, lists, tables
  • Add media — images, videos, audio
  • Create links — connect pages together (hyperlinks)
  • Build forms — text inputs, buttons, checkboxes for user input
  • Organize layouts — headers, navigation bars, footers, sections
  • Embed other content — maps, social media posts, iframes

What HTML Cannot Do

  • Cannot perform logic or calculations (needs JavaScript)
  • Cannot style pages beautifully (needs CSS)
  • Cannot store data permanently (needs backend/database)
  • Is not a programming language — it's a markup language

CSS — Cascading Style Sheets

CSS is the language that styles HTML. If HTML is the skeleton, CSS is the skin, clothes, and makeup — it controls how everything looks.

How It Works

CSS uses selectors to target HTML elements, then applies properties and values:

h1 {
  color: blue;
  font-size: 24px;
}

What CSS Can Do

  • Colors & backgrounds — text color, background images, gradients
  • Typography — fonts, sizes, spacing, alignment
  • Layout — grids, flexbox, positioning, margins, padding
  • Spacing — control the space inside and around elements
  • Responsive design — adapt layouts to different screen sizes
  • Animations — transitions, keyframe animations, hover effects
  • Borders & shadows — box shadows, text shadows, rounded corners
  • Visibility — show/hide elements, opacity

What CSS Cannot Do

  • Cannot create or add content (that's HTML)
  • Cannot perform logic or calculations
  • Cannot interact with users beyond hover effects (needs JavaScript)

JavaScript — The Programming Language of the Web

JavaScript (JS) is what makes web pages interactive and dynamic. If HTML is the skeleton and CSS is the appearance, JavaScript is the brain and muscles — it brings the page to life.

How It Works

JS runs in the browser and reacts to events, manipulates the page, and communicates with servers:

document.querySelector('.btn')
.addEventListener('click', () => {
  alert('Clicked!');
});

What JavaScript Can Do

  • React to user actions — clicks, typing, scrolling, mouse movement
  • Update content live — change text, images, styles without reloading
  • Validate forms — check inputs before submission
  • Handle data — store, retrieve, and manipulate information
  • Fetch data — load content from APIs/servers in the background
  • Create complex features — image sliders, dropdown menus, modals, tabs
  • Build full applications — games, chat apps, interactive tools
  • Run on servers too — via Node.js, not just browsers

What JavaScript Cannot Do (in the browser)

  • Cannot access files on your computer directly (security restriction)
  • Cannot read data from other websites unless they allow it
  • Cannot run by itself — it needs a browser or Node.js environment
